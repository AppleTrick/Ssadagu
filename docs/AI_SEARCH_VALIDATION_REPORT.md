# AI 자연어 검색 검증 리포트 (2026-06-22)

실제 운영 컨테이너(`ssadagu-back`)에 테스트 계정과 16개 상품을 등록하고, 실 서비스 엔드포인트(`/api/v1/products`, `/api/v1/products/search`)를 직접 호출해 측정했다. 작업 경과는 `docs/history/20260622.md` 참고.

## 1. 측정 결과

### 1-1. 복합조건 검색 정확도

검색어: **"100만원 이하 검정 맥북 프로 미개봉"**

테스트 상품 6건(정답 1 + 가격/색상/조건/모델이 각각 하나씩 다른 오답 5건)을 설계해 등록.

| 항목 | 기존 키워드 검색 | AI 검색 |
|---|---|---|
| 정답(95만원·스페이스블랙·미개봉) 매칭 | 0/1 | 1/1 |
| 오답 5건(가격초과/색상다름/조건다름/모델다름/색상+조건다름) 오인식 | - | 0/5 (전부 정확히 제외) |

**정밀도 100%, 재현율 100%.**

### 1-2. 표현 변형 매칭 (동의어/번역 변형 18건)

같은 상품을 다른 표현(한글/영어/색상 변형)으로 검색했을 때 매칭되는지 측정.

| | 기존 키워드 검색 | AI 검색 (버그 수정 전) | AI 검색 (버그 수정 후) |
|---|---|---|---|
| 매칭 건수 | 0/18 | 5/18 (28%) | **15/18 (83%)** |

버그 수정 전후 차이는 아래 2-1, 2-2의 수정 때문. 남은 3건(나이키 화이트/흰색 검색 2건, 노스페이스 검정 롱패딩 1건)은 미해결 — 원인 미파악.

### 1-3. 폴백/검증 로직 실제 작동 횟수

- 상품 16건 동시 등록 시 Groq `groq/compound`의 on-demand TPM 한도(8000) 초과로 **14/16건이 즉시 429 rate limit** → `AIMetadataService`가 예외를 잡아 빈 메타데이터로 폴백 처리됨을 로그로 확인
- 재시도 과정에서도 개별 상품마다 2~5회씩 추가 실패→폴백이 반복됨
- → "LLM 실패 시 안전하게 폴백한다"는 설계가 가정이 아니라 실제 운영 중 빈번하게 발동하는 안전장치임을 확인

## 2. 발견한 버그

### 2-1. 색상 정규화 불일치 (수정 완료)
- 등록 단계는 색상을 13개 화이트리스트로만 저장(`canonicalColors`), 검색 단계는 화이트리스트 없이 원문 그대로 추출(`colorAliases`)
- 예: 등록 시 "미드나이트" → "검정"으로 저장. 검색 시 "미드나이트" 그대로 검색 → 불일치로 매칭 실패
- 수정: 등록 시 실제 표현 + 화이트리스트 값을 함께 저장 (`AIMetadataService.extractMetadataFields`)

### 2-2. 카테고리 코드 불일치 (수정 완료)
- 프론트엔드 실제 카테고리 코드(`ELEC/CLOT/BOOK/FURN/SPOR/BEAU/FOOD/KID/HOBB/ETC`)와 `AIMetadataService`가 가정한 코드(`DIGITAL/FASHION/FURNITURE`)가 전혀 다름
- 검색 단계에서 LLM이 category를 자유 텍스트("전자기기")로 추출하면 `category_code = ?` exact match에 걸려 brand/product가 맞아도 결과 0건
- 수정: `buildExtraHint()` 코드 교정 + 검색 단계 category에 화이트리스트 검증 추가

### 2-3. 상품 이미지 URL이 가짜 주소 (수정 완료, 인프라 영향 있음)
- `S3Service.uploadImage()`가 실제 AWS S3 URL 형식을 반환했지만 저장소는 MinIO → 존재하지 않는 URL
- 수정 과정에서 실제 MinIO 공개 라우팅(`s3.changhee.dev`)이 `tailscale-only`로 막혀 있고, 버킷도 비공개였다는 것도 함께 발견
- **인프라 변경**: `tailscale-only` 미들웨어 제거 + 버킷 `download`(공개 읽기) 정책 적용 → 누구나 상품 이미지 URL에 직접 접근 가능해짐 (의도된 변경, 사용자 확인 받음)

### 2-4. (미해결) 나이키/노스페이스 일부 검색 실패
- "나이키 화이트 스니커즈", "나이키 흰색 운동화", "노스페이스 검정 롱패딩" 3건이 여전히 매칭 안 됨
- 색상·카테고리 버그 수정 후에도 남아있어 다른 원인(브랜드/제품명 추출 방식 차이 등) 추정, 원인 분석 필요

## 3. 부수적으로 발생한 이슈

- DB 정리 작업 중 테스트 계정뿐 아니라 실사용 계정(`arrans@naver.com` 등)까지 함께 삭제되어 웹사이트 로그인이 끊김 (사용자 확인 후 재가입으로 처리)
- `groq/compound`는 비전(이미지) 입력 미지원 → 이미지 포함 요청은 별도 비전 모델(`meta-llama/llama-4-scout-17b-16e-instruct`)로 자동 분기하도록 처리
