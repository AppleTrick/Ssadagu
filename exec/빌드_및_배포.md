## 2. 빌드 시 사용되는 환경 변수 상세 기재

프로젝트 빌드 및 실행을 위해 아래의 환경 변수가 각각의 `.env` 파일(또는 Jenkins Credentials)에 설정되어야 합니다.

### ⚙️ 외부 주입 환경 변수 (Jenkins)
빌드 파이프라인(`Jenkinsfile`) 실행 시 **Jenkins Credentials(Secret file)** 로부터 아래 파일들을 주입 받습니다.
- `BACKEND_ENV_FILE`: Backend 용 `.env` 파일 (위치: `./Ssadagu-Backend/.env`)
- `FRONTEND_ENV_FILE`: Frontend 용 `.env.local` 파일 (위치: `./Ssadagu-Frontend/.env.local`)

### 📁 Backend 핵심 환경 변수 (`Ssadagu-Backend/.env`)
| 변수명 | 설명 | 예시 / 기본값 |
|---|---|---|
| `MYSQL_URL` | MySQL 데이터베이스 접속 URL | `jdbc:mysql://[IP]:3306/ssadagu` |
| `MYSQL_USERNAME` | MySQL 접속 계정명 | `root` 등 |
| `MYSQL_PASSWORD` | MySQL 접속 비밀번호 | `password` |
| `JWT_SECRET` | JWT 서명용 키 (최소 256bit 이상) | `your_very_long_jwt_secret_key_...` |
| `SSAFY_API_KEY` | SSAFY 금융망 API 인증 키 | - |
| `SSAFY_API_USER_KEY` | SSAFY 금융망 API 유저 키 | - |
| `AWS_ACCESS_KEY` | 이미지 업로드용 AWS S3 Access Key | - |
| `AWS_SECRET_KEY` | 이미지 업로드용 AWS S3 Secret Key | - |
| `AWS_REGION` | AWS S3 리전 | `ap-northeast-2` |
| `S3_BUCKET_NAME` | AWS S3 버킷명 | `ssadagu-bucket` |
| `GMS_API_KEY` | GMS API 구동용 Key | - |

*(주의: 로컬 테스트 시 `application.properties` 에서도 참조 가능)*

### 📁 Docker Compose 환경 변수 (`docker-compose.yml`)
- `SPRING_PROFILES_ACTIVE=prod`: Backend 운영 프로파일 활성화
- `TZ=Asia/Seoul`: 모든 컨테이너 타임존 설정

## 3. 배포 시 특이사항 기재

1. **빌드 도구 분리 및 캐싱 전략**
   - **Backend**: `gradlew bootJar` 명령을 통해 사전 빌드(Build Stage) 후, 실행용 경량 이미지(`eclipse-temurin:21-jre-jammy`)에 배포하여 최적화를 수행합니다.
   - **Frontend**: `pnpm` 기반으로 `Next.js standalone` 모드를 사용하여 필요한 산출물만 추출하여 배포 용량을 줄였습니다. Docker 빌드 이전에 `.env` / `.env.local` 의 파일 복사가 성공적으로 이루어져야 빌드가 진행됩니다.

2. **자동화된 배포 프로세스 (Jenkins 파이프라인)**
   - GitLab 레포지토리의 `dev` 브랜치에 코드가 `push` 또는 `merge` 되면, Jenkins의 호스팅 파이프라인 과정(`Jenkinsfile`)이 발동됩니다.
   - 빌드 과정이 끝나면 최종적으로 `docker-compose -p s14p21a202 up -d backend frontend` 명령을 통해 운영서버에 배포가 적용되며, `nginx -s reload`로 무중단에 가까운 서버 갱신을 실행합니다.

3. **HTTPS 및 SSL 리버스 프록시**
   - Nginx와 Certbot (Let's Encrypt) 컨테이너를 함께 띄어 인증서를 발급 및 관리(`certbot renew` 주기적 실행)합니다. Nginx 측에서 80 포트를 443으로 리디렉션 처리하고 리버스 프록시를 통해 내부 Backend(8080)와 Frontend(3000)를 중계합니다.
   - 처음 배포 시 Certbot 볼륨 마운트 에러가 발생하지 않도록 초기 인증서 세팅에 유의해야 합니다.

4. **ELK 스택 (Elasticsearch, Logstash, Kibana) + Filebeat**
   - 전체 로그 수집용 구성을 위해 `docker-compose`로 함께 기동합니다. Elasticsearch의 메모리 제한(`ES_JAVA_OPTS=-Xms4g -Xmx4g`)이 설정되어 있으므로 호스트 서버(EC2)의 RAM 여유(최소 8GB 이상)가 필수적입니다.

## 4. DB 접속 정보 등 프로젝트에 활용되는 주요 속성 파일 목록

| 파일 위치 / 파일명 | 용도 및 설명 | 포함 내용 |
|---|---|---|
| `/Ssadagu-Backend/src/main/resources/application.properties` | Spring Boot 기본 애플리케이션 프라퍼티 | Jpa 관리 모드(ddl-auto), 로깅 포맷 설정, S3 / 금융망 API 엔드포인트 세팅, .env 치환 변수 목록 정의 |
| `/Ssadagu-Backend/.env` | Backend 민감 데이터 정의 (중요 자산, gitignore 처리) | DB 접속 계정, JWT 인증키, AWS / GMS 보안 키 등 |
| `/Ssadagu-Frontend/.env.local` | Frontend 빌드 민감 데이터 (중요 자산, gitignore 처리) | 클라이언트 통신 서버 URL 및 API 키 목록 |
| `/docker-compose.yml` | 운영 환경 인프라 정의 파일 | Nginx, Certbot, Jenkins, Backend, Frontend, ELK Stack 컨테이너 설정 |
| `/Ssadagu-Backend/schema.sql` | DB 스키마(DDL) 파일 | Users, Transactions, Products 등 전체 프로젝트 서비스의 주요 테이블 생성 쿼리 |

*(자세한 DDL 및 테이블간 관계는 `/Ssadagu-Backend/schema.sql` 에서 직접 확인할 수 있습니다.)*
